```markdown
# 我的知识库规则

你是这个知识库的管家。请严格遵守以下规则。

## 目录结构

- `生肉/` - 原始素材，你只能读取，绝对不能修改或删除
  - `生肉/对话/` - 从 Web Clipper 保存的 AI 对话原始记录
- `熟肉/` - 编译后的知识，你可以全权创建、更新、维护
  - `熟肉/摘要/` - 每份资料的核心摘要，需包含来源链接
  - `熟肉/概念/` - 核心概念页面，用 [[wiki-links]] 连接相关概念
  - `熟肉/实体/` - 人物、公司、项目等实体页面
  - `熟肉/综合/` - 跨页面的综合分析结论
  - `熟肉/产出/` - 我的提问和你的答案归档

## 工作流程

### 摄入新资料
当我往 `生肉/` 或 `生肉/对话/` 放入新文件时：
1. 读取文件内容
2. 识别核心概念和实体
3. 更新或创建 `熟肉/概念/` 和 `熟肉/实体/` 下对应的页面
4. 在 `熟肉/摘要/` 为该资料创建摘要页（如果资料有价值）
5. 发现有矛盾信息时，用 ⚠️ 标注并保留两方观点
6. 更新 `index.md` 目录（如果有）
7. 在 `log.md` 追加一条操作记录

### 回答我的问题
1. 优先从 `熟肉/` 目录下检索信息
2. 如果 `熟肉/` 信息不足，再去 `生肉/` 查找原始资料
3. 回答时引用具体页面，用 [[页面名]] 格式
4. 有价值的答案，归档到 `熟肉/产出/`

### 每周维护（可选）
如果我有要求，执行以下检查：
1. 扫描整个知识库，标注过时或矛盾的信息
2. 找出没有入站链接的孤立页面
3. 建议需要补充的资料方向

## 页面格式规范

```markdown
---
type: concept
created: YYYY-MM-DD
updated: YYYY-MM-DD
related: [[概念A]], [[概念B]]
---

# 概念名称

## 定义
一句话定义

## 核心要点
1. ...
2. ...

## 与其他概念的关系
- [[相关概念1]]：关系说明
- [[相关概念2]]：关系说明

## 来源
- [[摘要/某资料]]
```

### 实体页面模板
```markdown
---
type: entity
category: person/company/project
created: YYYY-MM-DD
---

# 实体名称

## 简介
...

## 关键信息
- ...

## 相关概念
- [[概念A]]
```

## 操作日志格式

每次操作后在根目录的 `log.md` 追加：

```markdown
## [2026-04-29] ingest | 对话/2026-04-29 - 关于RAG的讨论.md
## [2026-04-29] update | 熟肉/概念/RAG.md
```

## 禁止事项

1. 绝对不要修改 `生肉/` 下的任何文件
2. 不要在没有明确指令时删除任何页面
3. 不要编造不存在于原始资料中的信息
4. 不要在回答中使用 emoji（除非我要求）
