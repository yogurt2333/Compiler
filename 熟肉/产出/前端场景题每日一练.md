# 前端场景题每日一练

## 是什么

每天一道前端面试场景题，模拟面试官追问，训练「把知道的东西说出来」的能力。

## 方法论

所有场景题按 **三段论** 回答：

```
1. 结论/定性（1句话）
2. 拆解分析（第一…第二…第三…）
3. 方案/总结（所以我的方案是…）
```

## 练习记录

### Day 1 — 2026-05-06

**场景：** AI 聊天产品，历史消息越多页面越卡，打字都延迟了。

**我的回答：** 内存泄漏 → diff算法问题...

**Hermes 点评：** 方向对了，但回答太短。面试官要的是「排查思路 + 根因分析 + 解决方案」三段式展开。核心是：输入框和消息列表在同个组件，每次按键触发全量 re-render，几千条消息的 VNode 创建本身就有开销。

**标准三段论答案：**
> 结论：这是 VNode 创建开销问题，不是内存泄漏。
> 拆解：第一，输入框和列表在同个组件，每次按键触发了全量 re-render；第二，虽然 diff 没动 DOM，但几千条消息的 VNode 创建本身就有性能开销；第三，如果每条消息还有 computed/watch，成本更高。
> 方案：把输入框抽成独立子组件，配合虚拟滚动只渲染可视区域。

### Day 2 — 2026-05-07

**场景：** 报表系统，用户快速连续点查询时，前一次请求回来后一次请求没回来，旧数据覆盖了新数据。

**我的回答：** 用 Promise 链式调用 + 防抖，把连点合成一次请求。

**Hermes 点评：** 链式调用思路对——保证请求顺序，但报表查询本身慢（3-5s），连点N次就是N倍等待时间，用户体验反而不如直接用防抖。

**标准三段论答案：**
> 结论：应该用「防抖 + AbortController 取消」的组合方案，不是纯链式。
> 拆解：第一，链式调用不解决连点效率问题，用户等太久；第二，防抖处理连点行为（300ms后发请求），把N次合并成1次；第三，防抖仍需搭配 AbortController 取消上一个请求，防止防抖间隔内慢请求覆盖新数据。
> 方案：`debounce(300ms) + AbortController`，用户停手后发最新请求，同时 abort 上一个，保证展示的永远是最新数据。

**延伸思路：** 
- 如果需要同时对比两次不同条件的报表 → 多 Tab 方案，每个 Tab 独立维护请求
- 写操作的场景（表单提交）→ 链式调用更合适，不丢数据

### Day 3 — 2026-05-10

**场景：** 用 index 遍历列表时删除元素，为什么会串位？LeetCode 88 合并有序数组的变体。

**我的回答：** `pop(i)` 删除元素后后面的左移，`i += 1` 继续递增导致跳过了原本位于 `i+1` 位置的元素。

**Hermes 点评：** 原理说对了，但可以再展开变体题（合并有序数组从前往后填会覆盖未处理元素）和统一解法——倒序遍历/推导式。

**标准三段论答案：**
> 结论：遍历时用 index 删除元素会串位，因为 `pop(i)` 后元素左移 + `i` 递增导致跳位。
> 拆解：第一，`pop(i)` 删除后后面所有元素左移一位；第二，`i += 1` 继续递增，跳过了原 `i+1` 位置的元素；第三，LeetCode 88 合并有序数组是同一类问题——从前往后填覆盖了未处理的元素。
> 方案：原地改数组先想倒着走；能写推导式不碰 index 手。统一口诀：「倒序遍历 / 推导式」。

**延伸思考：**
- 推导式（列表/字典/集合）是声明式写法，和深浅拷贝是不同维度的概念（一个管「怎么生成」，一个管「怎么复制」）

### Day 4 — 2026-05-12

**场景：** 多步骤表单向导（注册/配置流程），5 个步骤，用户可以来回切换。当前实现每切一步就 mount/unmount 组件，导致回到上一步时之前填的数据全丢了。

**我的回答：** 持久化，扔到 localStorage 里。

**Hermes 点评：** 方向正确——解耦数据与组件生命周期。但面试官还会追问多 Tab 冲突、敏感信息安全、步骤联动校验等进阶场景。

**标准三段论答案：**
> 结论：用状态提升 + 中间层缓存（localStorage / IndexedDB），配合 keep-alive 缓存组件实例。
> 拆解：第一，问题根因是 Vue 生命周期 —— 切步骤销毁组件 → 局部 data 丢失；第二，三种方案对比——localStorage 手动读写（灵活但代码多）、keep-alive（简单但内存占）、Pinia+持久化插件（推荐，自动栓数据到 localStorage）；第三，进阶考量——多 Tab 同步监听 `storage` 事件、敏感信息用 `crypto.subtle` 加密、步骤联动校验通过 store getter 动态生成。
> 方案：**Pinia + pinia-plugin-persistedstate** 自动持久化到 localStorage，配合 `<keep-alive>` 包裹步骤组件。
