# 2026-05-04 Hermes 记忆系统架构

## 核心内容

Hermes Agent 的记忆系统通过四层分工协作，让 AI 既能随时调取核心信息，又能按需搜索历史对话。

## 四层架构

1. **提示词记忆（热记忆）**: `MEMORY.md`（~800 tokens，项目约定）和 `USER.md`（~500 tokens，用户偏好），AI 始终可访问，但变更为快照式，下次会话生效
2. **会话存档（冷记忆）**: SQLite 存储完整历史，AI 通过 `session_search` 工具按需关键词检索
3. **技能记忆（程序性记忆）**: 完成复杂任务后自动生成模板 Skill，下次类似任务直接调用，实现自我进化
4. **外部提供商（可选）**: 知识图谱（Hindsight）或纯 SQLite（Holographic）增强检索

## 容量与操作

- `MEMORY.md` 上限 2,200 字符，`USER.md` 上限 1,375 字符
- 内置 `memory` 工具支持 add / replace / remove 三个动作
- 超出限制时 AI 必须整合或删除旧条目

## 扩容方案
- **Hindsight**: 知识图谱，极致准确率
- **Holographic**: 纯 SQLite，零配置本地增强
