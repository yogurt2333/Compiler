# Hallucination（幻觉）

## 定义

LLM 在生成时「编造事实」的现象。不是 bug，而是 Transformer 统计预测本质的副产品——模型不知道「正确」还是「错误」，只知道「统计上最可能的下一个 token」。

## 四种变体

| 类型 | 表现 | 原因 |
|:----|:-----|:-----|
| **知识幻觉** | 编造不存在的事实 | 训练数据里没有或稀有 |
| **记忆幻觉** | 记错上下文信息 | 长上下文丢失/注意力分散 |
| **逻辑幻觉** | 自相矛盾 | 模型没有形式逻辑推理能力 |
| **关联事实幻觉** | 把 A 的属性安到 B 上 | 实体关联混淆 |

## 统一解决方案

1. **RAG（检索增强生成）** — 生成前先检索，把原文放上下文里当参考
2. **结构化输出** — JSON Schema / Regex 约束输出格式
3. **温度控制** — 降低 temperature 减少随机性
4. **Few-shot 示例** — 给模型提供正确输出的样例
5. **Chain-of-Thought** — 让模型先推理再输出，减少统计跳跃
