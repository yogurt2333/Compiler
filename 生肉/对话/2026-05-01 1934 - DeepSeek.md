## 📦 一、部署流程

### 1. ECS 环境准备
- 操作系统：Ubuntu 22.04 / Alibaba Cloud Linux 3
- 配置要求：内存 ≥ 2GB（推荐 4GB）
- 安全组规则：
  | 端口 | 用途 | 授权对象 |
  |------|------|----------|
  | 22 | SSH 远程连接 | 你的家庭/公司公网 IP |
  | 18789 | Hermes WebUI | 0.0.0.0/0 |

> ⚠️ 22 端口务必限制来源 IP，避免被暴力破解。

### 2. 安装 Hermes Agent
```bash
# 国内推荐使用镜像加速
curl -fsSL https://res1.hermesagent.org.cn/install.sh | bash
source ~/.bashrc
hermes --version   # 验证安装
```

### 3. 配置大模型（DeepSeek V4 Flash）
```bash
hermes config set model.provider deepseek
hermes config set model.base_url https://api.deepseek.com
hermes config set model.api_key sk-你的Key
hermes config set model.default deepseek-v4-flash
```

> 💡 Flash 版本响应更快（毫秒级）、价格更低（输出 2元/百万 token），适合日常对话和任务规划。

### 4. 验证模型连通性
```bash
hermes doctor   # 应显示 ✓ DeepSeek
hermes          # 测试对话，输入“你好”
```

### 5. 配置微信网关
```bash
hermes gateway setup
```
- 通道选择：`Weixin`
- 授权模式：`Use DM pairing approval`（个人使用可改为 `allow_all_users true`）
- 群聊处理：`Disable group chats`（官方不支持群聊）
- 安装服务：`User service`（实现开机自启）

**扫码登录**：
- 终端输出二维码链接 → 浏览器打开 → **微信小号**扫码
- 微信中会自动添加「**微信 ClawBot**」联系人

**配对批准**（首次需要）：
```bash
hermes pairing list
hermes pairing approve weixin [CODE]
```

### 6. 后台保活（systemd user service）
```bash
# 启用用户服务开机自启
sudo loginctl enable-linger $USER
```

> 选 User service 后建议执行此命令，确保服务器重启后网关自动运行。

## 🔄 四、下一步：Obsidian 同步方案

### 推荐方案：阿里云 OSS + Remotely Save（全自动双向同步）

**1. PC 端配置**
- Obsidian 安装 Remotely Save 插件
- 阿里云创建 OSS Bucket（私有读写）
- 获取 AccessKey ID/Secret
- 配置插件：Remote URL = `https://<bucket>.oss-cn-hangzhou.aliyuncs.com`

**2. ECS 端配置**
```bash
pip3 install oss2
ossutil config  # 配置 AccessKey
ossutil sync oss://obsidian-notes/ ~/obsidian-vault/ --update

# 设置定时同步（每小时）
crontab -e
0 * * * * /usr/local/bin/ossutil sync oss://obsidian-notes/ ~/obsidian-vault/ --update
```

**3. 权限设置**
```bash
sudo chown -R root:root ~/obsidian-vault
sudo chmod -R 755 ~/obsidian-vault
```

> 备选方案：Git + SSH（适合习惯手动 push 的用户）

## ⚠️ 六、踩坑与避坑

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `hermes: command not found` | PATH 未更新 | `source ~/.bashrc` 或重装 |
| Base URL 填错 | 误把 API Key 填到 Base URL | 正确：`https://api.deepseek.com` |
| API 401 错误 | Key 无效/账户余额不足 | 去控制台生成新 Key 并充值 |
| 微信无回复 | 未批准配对 | `hermes pairing approve` |
| 22 端口暴露风险 | 未限制来源 IP | 安全组只允许自己 IP |
