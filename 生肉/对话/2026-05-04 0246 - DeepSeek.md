## Obsidian Git 插件配置与使用记录

### 1. 环境准备
- 本地已有 Git 仓库（翻译文档所在的仓库）
- 仓库已关联 GitHub 远程地址
- 已完成 `git pull`，认证和网络连接正常

### 2. 插件安装与启用
- 在 Obsidian 社区插件市场中搜索并安装 **Obsidian Git**
- 启用插件

### 3. 手动验证
- 执行命令 `Git: Push`（命令面板：`Ctrl/Cmd + P` → 搜索 push）
- 验证通过，无认证错误 —— 说明之前配置的 GitHub 认证有效

### 4. 配置自动推送（核心目标）
在插件设置中完成以下配置：

| 设置项 | 操作 |
|--------|------|
| `Split automatic commit and push timers` | ✅ 勾选（分开控制提交和推送） |
| `Auto push interval (minutes)` | 设置为目标间隔（如 30 或 60 分钟） |

> 说明：勾选分离控制后，可独立设置提交和推送的间隔；本场景核心需求是**定时 push**，提交间隔可按需调整或沿用默认。

### 5. 最终效果
- 插件会按设定的时间间隔**自动 push** 本地变更到 GitHub
- 无需手动执行 `git push`

---

**一句话总结**：在 Obsidian 中安装 Git 插件 → 验证 Push 可用 → 开启自动 Push 定时器。
