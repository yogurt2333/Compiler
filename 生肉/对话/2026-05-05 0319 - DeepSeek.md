## 学习内容

### 第1节：aiohttp — Python 版 axios
- `async with aiohttp.ClientSession() as session` — 创建浏览器会话，用完自动清理
- `async with session.get(url) as resp` — 发 GET 请求
- `async with session.post(url, json=..., headers=...)` — 发 POST + JSON
- `await resp.json()` — 读取 JSON 响应
- `aiohttp` 类比前端 `axios`，`async/await` 类比 JS 的 async/await

### 疑问解答
- **为什么三个都要 async？** `async def` 声明异步函数，`async with session` 等浏览器准备好，`async with session.get` 等服务器响应，`await resp.json()` 等读完响应体
- **`as resp` 和 `as session` 是什么？** 上下文管理器赋值，等价于 `const resp = ...`，但自带用完自动清理
- **事件循环是什么？** Python 的"任务调度员"，遇到 `await` 就挂起去干别的，类比 JS 的事件循环（微任务/宏任务）
- **Jupyter 的坑：** 不能 `asyncio.run()`，直接 `await` 就行

### 第2节：pydantic — 数据模型 + 运行时校验
- 定义：`class X(BaseModel): name: str`
- 取值：`user.name`（属性式，不是 `user["name"]`）
- 自动校验类型，传错就抛异常
- `.model_json_schema()` 一键生成 JSON Schema 给 LLM
- `.model_dump()` / `.model_dump_json()` 序列化
- 与 TypedDict 区别：TypedDict 只有类型提示（编译时），pydantic 有运行时校验 + 自动类型转换

### DeepSeek API 调用示例
- 用 aiohttp POST 到 `https://api.deepseek.com/v1/chat/completions`
- Header：`Authorization: Bearer sk-xxx`
- Body：`model: deepseek-chat` + `messages` 列表
- 从 `result["choices"][0]["message"]["content"]` 取回复
